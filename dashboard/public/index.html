Point a wallet here

Electron Cash, Sparrow and other wallets connect over the Electrum protocol. Use the SSL endpoint — most wallets refuse plain TCP.

—
—
In Electron Cash: Network → switch to manual, add the server above. The certificate is self-signed, so accept it on first connect.

Wallet serving is off

This node runs in pruned mode, so it doesn't keep the full history an Electrum server needs. Wallets can't sync against it.

Want wallets to connect? Switch to Full mode under Storage below. That keeps the whole chain and turns the Electrum server back on.

Node RPC — for apps

Fulcrum and SoloStrike Cash connect to this node over JSON-RPC automatically. Miners do not point here — install SoloStrike Cash and point your ASIC at its stratum port instead.

The RPC port is not published to your network by default. Expose it deliberately in the app's compose file if an external app needs it, and change the default credentials first.

Tor

Run your node as a Tor hidden service. Onion add-on keeps fast clearnet peers and adds a .onion address; All traffic routes every connection through Tor (more private, slower).

Applying restarts the node (about 30–60s); the dashboard reconnects on its own.
Storage mode

Full

Active

Keeps the entire chain and serves wallets through the Electrum server. Uses the most disk — hundreds of GB and growing.

—on disk
—wallet server
  1. In docker-compose.yml, set the mode line to -txindex=1.
  2. Restore the fulcrum service if you removed it.
  3. Restart the app.
⚠Going from pruned to full re-downloads the chain from scratch — the discarded blocks have to come back.

Pruned

Active

Validates every block but discards old ones past a size cap. Small disk, RPC-only — no wallet server.

—size cap
—retained from
  1. In docker-compose.yml, change the mode line to -prune=5500 (MB to keep, minimum 550).
  2. Remove the fulcrum service — it can't index a pruned chain.
  3. Restart the app.
⚠Pruning is one-way for the existing data. Re-enabling full history later means a fresh sync.
